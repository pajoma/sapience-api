    • Typical scenarios for semantic annotations are semantic integration of data, and semantic validation of workflows. We have prepared a coupled of examples for Web services, including WSDL files and a capabilities- and process-description of an OGC Web Processing Service (WPS). This following link (original source) shows how a semantic reference is injected into the interface-definition of a WSDL 2.0 Web service:

    http://semantic-proxy.appspot.com/api/get?sid=6bbf18d0

    The SAWDL standard can be used for semantic annotations of certain WSDL elements (interface, operations, faults, inputs and outputs) and the XML schema in the types-section. The following URL (using the same source as in the first example) illustrates how the new attribute is dynamically injected into an existing schema definition:

    http://semantic-proxy.appspot.com/api/get?sid=a3c919de

    But semantic annotations are not only additional attributes, but in many cases complete new elements. The following is an example where a new element is injected in a WSDL 1.1 document:

    http://semantic-proxy.appspot.com/api/get?sid=6e3a18f2

    The Web service description standards specify the means to add references to external vocabularies. The OGC WPS, for example, makes use of the OWS Common standard. In this case, we make use of the metadata-field and the XLink-standard to add the references. The following example injects a Metadata-element in the Process-listing.

    http://semantic-proxy.appspot.com/api/get?sid=8f811d89
